2.16 Основы алгоритмов ч3.
Приводим нашу реализацию ArrayList к его идейному вдохновителю из стандартной библиотеки Java.

Для упрощения работы следует проводить с той реализацией, которая обрабатывает числа, а не строки.

1. Реализовать приватный метод grow, который будет отвечать за расширение массива-хранилища в 1,5 раза в ситуации, когда место закончилось.
2. Добавить проверку на заполненность в метод add и, если массив заполнен, расширить его.
3. Изменить реализацию сортировки на рекурсивную из последней шпаргалки.

    Выбор конкретной сортировки лежит на вас.


Повышенная сложность:

1. Добавить приватный метод resize, который будет уменьшать массив-хранилище на треть в тех ситуациях, когда он заполнен меньше, чем на половину.
2. Добавить проверку на необходимость вызова resize в метод remove.

- Подсказки
    1. Методы grow и resize должны создавать новый массив большей/меньшей размерности и копировать туда имеющиеся элементы.
    2. Методы grow и resize не должны быть добавлены в интерфейс, т.к. являются приватными.
